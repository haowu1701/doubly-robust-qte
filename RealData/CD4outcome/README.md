# CD4 Outcome Real-Data Analysis

This directory contains reproducible CD4 outcome analysis code using a synthetic dataset. The synthetic data are included so that the public code can be run without distributing private patient-level records.

## Files

- `MQ_PTE_CD4.Rmd`: marginal CD4 outcome analysis. This file estimates marginal quantile treatment effects (QTE) and probability treatment effects (PTE) using AIPW-CPM approaches.
- `CQ_PTE_CD4.Rmd`: conditional CD4 outcome analysis by sex. This file estimates sex-specific conditional QTE and PTE curves using the same AIPW-CPM framework.
- `Helper_functions.R`: helper functions used by the R Markdown files, including quantile inversion and conditional CDF evaluation from two outcome models.
- `data_model_cd4_YA_synthetic.RData`: synthetic analysis dataset used by the R Markdown files. It contains an object named `data_model`.
- `generate_synthetic_cd4_data.R`: maintainer script for regenerating the synthetic dataset from the private local data file.

## Data Availability

The original analysis data file, `data_model_cd4_YA.RData`, contains real patient-level data and is not included in this repository.

The public R Markdown files load:

``` r
data_model_cd4_YA_synthetic.RData
```

The synthetic file contains only the variables used by the CD4 analysis:

``` r
c(
  "Y",
  "A",
  "age",
  "sex",
  "site",
  "route_infection",
  "prior_aids",
  "calendar_year",
  "months_toCD4measure",
  "cd4_baseline_tran",
  "singledl_rna_baseline_v_tran"
)
```

`generate_synthetic_cd4_data.R` requires the private local file `data_model_cd4_YA.RData`. Public users do not need to run it unless they have access to the private source data.

## Synthetic Data Generation

The synthetic dataset is generated by `generate_synthetic_cd4_data.R` as follows.

### Continuous Covariates

The continuous covariates are:

``` r
age
months_toCD4measure
cd4_baseline_tran
singledl_rna_baseline_v_tran
```

These variables are generated jointly from a multivariate normal approximation. The script estimates the mean vector and covariance matrix from the private data, generates correlated normal random variables using a Cholesky decomposition, and clips each generated variable to the private data's 1st-99th percentile range.

### Binary Covariates

The binary covariates are:

``` r
sex
prior_aids
```

Each binary covariate is generated from a Bernoulli distribution using the empirical event probability from the private data, with the probability clipped to `[0.05, 0.95]`.

### Categorical Covariates

The categorical covariates are:

``` r
site
route_infection
calendar_year
```

Each categorical covariate is sampled with replacement from its observed categories using empirical category frequencies. A small `+1` smoothing term is added to the category counts before converting them to probabilities. `calendar_year` is sampled as a discrete year, preserving integer values.

### Treatment Variable

The treatment variable is:

``` r
A
```

The script fits a logistic propensity model in the private data:

``` r
treatment ~ age + sex + site + route_infection + prior_aids +
  calendar_year + cd4_baseline_tran + singledl_rna_baseline_v_tran
```

The fitted model predicts treatment probabilities for the synthetic covariates. These probabilities are clipped to `[0.05, 0.95]`, and synthetic treatment values are drawn from Bernoulli distributions.

### Outcome Variable

The outcome variable is:

``` r
Y
```

The script fits an outcome regression in the private data:

``` r
log(Y) ~ A + age + sex + site + route_infection + prior_aids +
  calendar_year + cd4_baseline_tran + singledl_rna_baseline_v_tran +
  months_toCD4measure
```

Predicted log-scale outcomes are computed for the synthetic covariates, residual noise is added using the residual standard deviation from the fitted model, and the values are transformed back to the original CD4 scale using `exp()`. The generated outcomes are clipped to the private data's 1st-99th percentile range.

## Important Note

The synthetic data do not contain real patient records. Numerical results, fitted models, tables, and figures generated from the synthetic data are for code demonstration only and should not be interpreted as study findings.
